Queria saber se o DNS é um servidor físico ou uma espécie de software, e gostaria de saber se o DNS se encontra no mesmo ''servidor/local'' para qual foi o meu pedido. ex: eu faço um pedido de www.google.com.br, o DNS q identifica esse meu pedido fica no mesmo servidor que me respondeu com o HTML, CSS, JS. ​​

O DNS no inglês significa Domain Name System que, traduzido para o português significa Sistema de Nomes de Domínio. Permitindo que o usuário acesse sites e páginas através do endereço digitado no navegador. Que converte um nome de domínio em um endereço de IP (uma sequência numérica que dá a localização real de uma página na web) ao servidor em que o ele está hospedado!

Nesse link aqui, você pode dar uma aprofundada no assunto: https://tecnoblog.net/responde/o-que-e-dns/​​

Só reforçando, sim, ele fica em um servidor e nesse servidor que tem um tipo de banco de dados que guarda as informações de IP. Esse servidor tem um software específico que retornará as informações esperadas de sua requisição.

Então o DNS "traduz" o link do site para o endereço de ip do servidor no qual o site está hospedado?

Isso, o DNS converte um endereço entendível por humanos como uma "url", para um endereço de ip. Dá uma olhadinha aqui: https://tecnoblog.net/responde/o-que-e-dns/

Como que o google acha o site que eu quero msm escrevendo errado, tipo em vez de escrever programador eu escrever prograbador?

Basicamente, o Google funciona utilizando robôs (conhecidos como Googlebot, crawler ou spider), que vão de link em link identificando as páginas na internet, classificando e indexando em seus servidores. Então mesmo digitando errado, ele faz essa busca. Dá uma olhada depois neste link aqui: https://developers.google.com/search/docs/basics/h...

É possível um site ter mais de um DNS para acessar o único IP, exemplo, se eu digitar (e se esses sites existirem) "programadorbr.com.br" ou "progbr.com.br" ou "programabrasil.com.br" ele pode levar para o IP desse site para o cliente acessar? ou o DNS deve ser único e vinculado a um único IP? e domínio e DNS seriam a mesma coisa?

Você pode sim ter vários domínios para um ip

É possível criar um site e outro site ter o mesmo nome, o que acontece nesta situação? Quando for acessar o site, a computador será redirecionada para qual dos arquivos que se encontram no servidor?

Não há como isso ocorrer. O que pode acontecer é você ter subdomínios, que possibilitaria fazer algo do tipo, mas nesse caso cada rota iria enviar um arquivo específico.​​​​​​ ​​​​​​​

Então basicamente quando eu busco um endereço web (www), a requisição vai até o DNS que traduz o web para IP e manda pro servidor que está hospedando os arquivos, que consequentemente envia os arquivos pra mim. É isso?

Sim. Exatamente isso!

​Então, ele provavelmente está rodando num servidor local, então ao conectar na wifi dela, você está conectando na mesma rede que permite o acesso. Na verdade tanto faz a rede ser wifi ou por cabo, o importante é estar na mesma rede. DNS seria para o registro de servidores?

O DNS seria uma tradução entre a gente e a máquina, de uma forma simplificada. Ele serve pra traduzir o URL digitado por nós para o ip da máquina que está com os arquivos da página que queremos acessar.
