DÚVIDAS DOS ALUNOS

Se eu quisesse apenas trabalhar com frontend, e eu quisesse apenas estudar modulos especificos, funcionaria? ou eu preciso estudar todo o curso da msm forma? se sim, quais modulos me recomendaria?

Mesmo que você queira focar apenas no front, a recomendação é que você veja o curso todo na ordem que foi programado. Falo isso, pois há conceitos do back que também são importantes para você tenha breve noção de como funciona e assim controlar melhor as informações.

É possivel fazer um site apenas com backend?

Não. Apenas com backend você consegue entregar informaçõs mas não consegue formatá-las, esse é o papel do Front-end

Poderia trazer de forma escrita, o que é frontend e backend?

​Então, é de fundamental importância que faça breves resumos sobre o que entendeu de cada aula, com suas próprias palavras usando um bloco de notas, por exemplo, para fixar melhor os conceitos. 

O front-end é toda a parte que o cliente utiliza, botões, cards, funções para fazer algo no seu site e etc. 

O back-end é toda a lógica por trás dos panos, onde o cliente não vê. 

EX: digamos que você vai fazer um login, você colocará seus dados em um formulário, esse formulário está no frotn-end, você está interagindo com ele. Quando você clica para fazer o login, esses dados são enviados para um servidor (backend) que irá verificar se realmente há algum usuário com esses dados e se você pode ou não fazer esse login. 


Eu havia lido na internet que o front end era a parte visível do site, onde o usuário interage, e que o back end era a parte dos códigos, que não aparece na tela. Este conhecimento prévio que mencionei está equivocado? A diferença entre frontend e backend seria então apenas que um roda no meu computador e o outro roda no servidor remoto?​​​​​​​​​​

Tem alguns pontos certos e outros incompletos nessa interpretação. O Frontend é tudo que roda no seu computador e o Backend é tudo que roda em um computador remoto (servidor).

Porém nem tudo que não aparece na tela está rodando no servidor. Por exemplo, se você usa uma aplicação web que te permite fazer um desenho. Existe uma parte não visual, que é o código que calcula as coordenadas para desenhar na tela, mas que roda no seu computador e faz parte do front end. Se vc só quer poder desenhar e baixar o arquivo para seu computador, essa aplicação pode ser 100% front end. Se vc quiser salvar para que outra pessoa possa abrir com um link por exemplo, você irá precisar do backend para salvar a imagem no servidor e assim permitir que vc mesmo ou outras pessoas tenham acesso de outros computadores.

Posso dizer que o frontend seria o layout da pagina, o design dela e o backend seria o mecanismo dela o que faz realmente funcionar?

Seria mais ou menos isso. O front está ligado com toda a parte visível utilizável pelo cliente porém ele vai  além de ser somente o layout. O backend seria toda a lógica, onde o cliente não tem acesso, Isso ficará mais claro quando você chegar na parte de node.js onde você entenderá como funciona um servidor.
