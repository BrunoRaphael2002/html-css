DÚVIDAS DOS ALUNOS

O servidor de aplicações e de arquivos são exclusivos para cada tipo de pagina, tipo o servidor de aplicaçoes é apenas para paginas dinamicas e o de arquivos apenas pra paginas estaticas?

Na verdade, não. No servidor de aplicação você também pode mandar páginas estáticas, além disso, você pode fazer muitas outras coisas, como por exemplo APIs. No decorrer do curso cada ponto vai ficando mais claro.​​​

Então no caso, existem dois tipos de servidores? Um de arquivo simples quando é uma página estática e outro servidor de aplicação quando é uma página dinâmica? Está correto?

Não necessariamente dois tipos de servidores, o mesmo servidor pode "servir" tanto um quanto outro, porém quando contratamos este tipo de serviço pagamos mais caro pelo servidor com páginas dinâmicas, ou aplicações web. Mas não se preocupe muito com isso agora, o Igor fala sobre isso no início apenas para dar uma base, ao longo do curso será demonstrado bem cada um.

Pensei que quem estaria por trás das paginas dinamicas e o processamento e transformacao de dados seria o JAVASCRIPT do Front End, e nao o BackEnd, poderia me explicar melhor sobre isso?

Então, o javaScript ele atua tanto no back como no front, o que ele queria dizer com página dinâmica são páginas que recebem dados lá no back-end antes de enviar para o cliente, ou seja, nos populamos a página de maneira dinâmica. Tudo isso será explicado melhor no decorrer do curso. O JavaScript do front também pode puxar dados do back e deixar as coisas bem mais dinâmicas, além de toda funcionalidade que ele cria, porém, você verá que dependerá da necessidade.

Vai com calma, segue a ordem do curso que tudo isso vai ficando claro. Esse módulo um é bem introdutório, você verá melhor a respeito no decorrer dos módulos e aulas.

Se eu programo um sistema como o da Globo.com ou de uma loja , eu teria que ter em um mesmo sistema páginas do site para usuários do sistema inserirem as reportagens ou produtos e outras páginas para mostrar as reportagens ou produtos do site aos internautas?​​​​​​​​​

No momento do curso você não precisa esquentar para entender isso, mas você teria uma página administrativa para poder controlar as postagens novas no seu site, lá em node quando estiver lidando com sites dinâmicos você entenderá melhor.

​Estou com uma dúvida sobre página dinâmica: eu entendi que o servidor vai pegar as informações de um banco de dados, mas eu preciso enviar um "arquivo" banco de dados (similar aos .html, .css, .js) ou precisaria contratar algo além para isso?

Você não precisa enviar nada, o seu servidor fará a conexão com o seus dados no banco, que podem estar localmente, ou na nuvem, e conseguirá ler e passar esses dados que serão populados no template. No nosso caso utilizaremos o EJS, mas é assunto para bem mais para frente. Você não precisa entender profundamente isso no início, mas quando chegar lá e vê como tudo funciona, entenderá melhor tudo isso.

​O que é template e pra que serve?

​Então, o template nada mais é do que o modelo da página que será utilizado para criação de uma versão de uma página. Isso ocorre no exemplo do Igor quando ele mostra duas páginas totalmente iguais, porém com dados diferentes. Você não precisa esquentar com isso nesse primeiro momento porque isso será melhor abordado quando você estiver lidando com node.
