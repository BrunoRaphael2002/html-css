DÚVIDAS DOS ALUNOS

As tags dentro das Divs perdem o significado?

Não, a div seria uma forma de "englobar" os elementos. Vou tentar exemplificar, Pense na página principal do google, https://www.google.com.br/ 

Ali no centro você tem uma div, dentro dela você tem uma imagem, abaixo um input, depois dois botões um do lado do outro e por fim um texto que poderia ser até outra div.

Agora pensando na página do curso, você tem a barra de navegação em cima blz, depois na esquerda tem uma aba com as aulas certo? Essa aba seria uma div, ocuando a maior parte da tela tem o lugar onde há o vídeo, essa seria outra div, depois abaixo tem os comentários, que seria outra div, dentro dos comentários, cada comentário (esse "card") seria uma div, e assim por diante. A div é a tag mais usada nos códigos porque ela serve para agrupar um conjunto de elementos, e usamos elas para posicionar cada um desses grupos na página.

Gostaria de saber qual o intuito de ficar colocando Div dentro de Div a cada novo elemento no código? Se a Div não tem peso para o mecanismo de busca, desta forma não fica bagunçado o código? Só estou perguntando pois notei que em aulas futuras o Igor usa muito esse recurso.

É muito normal essa sua dúvida, principalmente por ainda estar bem no início do curso, mas geralmente colocamos uma div dentro de outra para poder ter algum resultado específico seja com alinhamento ou com estilo.

Por exemplo: Digamos que eu quero um quadrado que tenha uma imagem, e uma cor transparente que dê para vê a imagem de fundo. poderiamos colocar uma div dentro de outra, uma teria a imagem, já a externa teria a cor transparente.
