​Quer dizer que nao posso colocar id em 2 tags do mesmo tipo mas se for 2 tags de funcoes diferentes posso usar o id?

Então, o ID precisa identificar um único elemento na sua página, não importando se dois elementos tem a mesma tag ou não.

Então id repete uma unica vez e class pode-se repetir, mas qual a aplicabilidade disso? Poderia deixar somente o class, certo? Sem se preocupar se alguma id foi repetida ou não.

Poderia para muitas coisas. Mas existe alguns casos que vc deve usar ID. Por exemplo, imagine uma página de vendas muito longa. Lá no final vc tem a oferta em uma div. Vc pode colocar um id nessa div, e lá em cima, no menu, pode colocar um link para a oferta. Quando o usuário clicar nesse link, a página vai direto para o final onde tem a div com esse id. Nesse caso não daria para usar class pq vc quer identificar aquela única div. Ficaria algo como <a href="#id_da_div_de_oferta">Oferta</a>

​O Igor comenta que o id refere-se a somente um elemento e o class refere-se a vários elementos. Testei o id no primeiro parágrafo do texto e no último, utilizando a mesma cor para os dois e funcionou. Logo, o id funciona do mesmo modo que a class quando se refere a cores. Correto?

Não, pode parecer funcionar, vc ate pode fazer, mas não é recomentado de forma alguma, certamente usando o id em mais de um elemento isso vai te trazer algum tipo de problema, principalmente quando precisar buscar esse elemento pelo JS, ele só vai trazer um, então vc ate pode colocar o id em vários elementos, mas isso vai te trazer problemas não é legal.

​Eu posso usar id e class num mesmo elemento? Por exemplo, eu quero ter um link que quando eu clico nele, ele role a página até chegar em uma imagem específica da página (usando o id), mas também quero que por padrão todas as imagens do site tenham uma borda estilizada padronizada (aí eu uso o class)

​Sim vc pode usar os dois, mas se quer todas as imagens do site com um certo estilo, vc pode estilizar a tag img direto pelo CSS, não precisa ficar colocando a class na tag img.

​Eu posso usar a classe como id, tipo usar uma classe para identificar um único elemento, mesmo que ele não se repita?

​Vc até pode, só não sei se é interessante. No JS para buscar por ID usamos o getElementById, ele traz apenas um elemento, se usar um seletor de classe mesmo tento só um elemento, ele vai trazer uma lista, não é nada de mais porem eu usaria o ID por facilidade organização e padronização

​Usando o VS Code eu testei o id e o class pra ver se de fato o id era exclusivo, e ocorreu que ele tem o mesmo funcionamento do class, não sei o que houve mas consegui alterar a cor de mais elementos usando a mesma identificação no id.

​Então, no nosso mundinho da programação,se padronizou id como uso único... pode até responder da mesma forma, mas isso é extremamente não recomendado... o ideal é usar id apenas para um item e class para mais de um :) de uma lida aqui: https://developer.mozilla.org/pt-BR/docs/Web/HTML/...​​​​​​​​​​

​Estou com uma duvida em relação ao ID e o CLASS, quando é melhor usar o id e quando é melhor usar class??

​Idealmente a classe será usada para dar estilos quando você quiser repetir esses estilos, além disso um mesmo elemento pode receber diversas classes, ele vai absorver os estilos de todas elas. Já o id será para elementos que vão ser únicos na página, além disso o id será preferencialmente usado quando precisarmos selecionar um elemento dentro do html quando estivermos usando o javascript (nos módulos de javascript isso será demonstrado), ele irá facilitar muito a seleção dos elementos. Mas no momento não irá influenciar tanto um ou outro.

​Como funciona o class?

​Damos classes para os elementos para depois no css podermos dar estilos para eles, por exemplo se quisermos que um elemento tenha a cor do texto azul, damos uma classe para esse elemento, essa classe podemos chamar de qualquer nome que quisermos, após dar a classe ao elemento, no css indicamos quais os estilos que essa classe vai ter. Veja o exemplo, tenho uma tag "p" e quero que ela possua o texto com cor azul no html: <p class="txt">Aqui escrevo meu parágrafo </p> no css: .txt{ color: blue; } Veja que chamei arbitrariamente a classe de txt.
