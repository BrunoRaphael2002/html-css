​Tem como deixar a borda somente ao redor da palavra?​​​

Há algumas formas de fazer isso. Você poderia adicionar um display inline nesse elemento, fazendo com que ele não possa ter suas dimensões alteradas e ocupe o tamanho do texto, dessa forma a borda cobriria apenas o texto. Você poderia adicionar uma tag chamada "span", abraçando apenas o texto, e depois adicionar a border na tag span. A tag span funciona parecido com a div, mas ela tem um display inline, para você conseguir estilizar regiões específicas dentro de um texto. 

Dá uma olhada aqui: https://developer.mozilla.org/pt-BR/docs/Web/HTML/...

Estou aplicando todo conhecimento do curso numa pagina que estou criando. Criei uma pagina, onde coloco uma borda no titulo principal (h1), Minha intenção, é de deixar essa borda como se fosse uma caixa de texto mesmo, bem rente a fonte. Até consegui fazer, porém, quando redimensiono a pagina, a borda redimensiona junto. Como faço para manter essa borda estática, sem se mover conforme se altere os tamanhos da pagina?

Então o h1 possui como default display block, caso não tenha assistido as aulas de display ainda não se preocupe que o Igor irá falar sobre, mas isso significa que ele ocupa toda a linha por isso a borda ocupa tudo. Para fazer a borda ficar só ao redor da palavra em si você pode colocar uma tag span dentro do h1, e nela passar a border 

html 

h1> 

span class="myClass">Meu Titulo</span> 

/h1> 

css

.myClass{ 

border: solid 1px black; 

}​​​​

​Criar um arquivo css para a pagina principal e um ou vários arquivos css separado para as outras paginas pra melhor organização é uma pratica válida ou isso não se pratica?

​Então, o que você pode fazer é criar um CSS principal onde vão ter os estilos que irão se repetir ao longo das páginas, por exemplo se você tiver um layout de header e footer iguais em todas as páginas, as vezes você pode fazer uma classe "padrão" que vá se aplicar em diversas páginas, e ai criar outros arquivos para o que for mais específico de cada página.

Brincando com as formas de fazer, percebi que posso usar o border-radius pra colocar raios diferentes em cada canto do meu "quadrado", por assim dizer, caso coloque: border-radius: 50px 40px 30px 20px; cada canto terá um raio diferente. Posso fazer isso também com 2 valores, sendo o primeiro valor para os cantos superior esquerdo e inferior direito e o segundo valor para os cantos superior direito e inferior esquerdo. Da mesma forma com border-color e border-style posso utilizar a mesma sacada pra ficar mais sucinto, porém esses com 4, 3, 2 ou 1 valor. Utilizar esse método é mais convencional do que ficar discriminando "border-top-style", "border-left-style”?

Eu particularmente uso sempre em uma linha, inclusive para margin e paddgin por exemplo margin: 10px 0 0 15px; Mas não é "melhor" ou pior, vai de cada um mesmo. O bom só é seguir um padrão, fazer o código todo no mesmo estilo, ou usa tudo numa linha ou coloca cada um top, right, etc.

Minha dúvida é sobre o que vai aparecer na tela de quem acessar o site quando há borda. Eu reparei que quando coloca a borda em um elemento, fica um espaço em branco entre o limite da janela e a borda esquerda, por exemplo, esse espaço vai aparecer para quem acessar o site?

Você verá mais a frente sobre a box model e entenderá que o border vai abraçar o elemento em si. O espaçamento em branco a esquerda, por exemplo, seria um margin do próprio body. Você pode adicionar no seu css: 

body { 

margin: 0; 

} 

Com isso você verá que não terá mais aqueles afastamentos. Você sempre tem que ficar de olho nas propriedades dos elementos. Sobre como aparecerá na tela de quem entrar no site, sim aparecerá da mesma forma que você estilizar seu documento html.

Então, o shorthand do border, ele só compacta as propriedades border-width, border-style, border-color. O border radius seria uma propriedade a parte, por esse motivo não tem como colocar tudo junto.
