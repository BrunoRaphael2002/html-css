Qual é a ordem das propriedades de background no shorthand?​

A ordem das propriedades de background é: background-color, background-image, background-size, background-repeat, background-attachment, background-position 

Tem como colar um background atrás de uma imagem? por exemplo, no exercício de montar o currículo, tem como colocar algum fundo atrás da imagem que tem naquele código?

Sim tem como você colocar um background atrás de uma tag img, mas só faz sentido se estiver utilizando uma com fundo transparente. Você mesmo pode fazer o teste, verá que para imagens sem o fundo transparente, o background não irá aparecer, mas para imagens com fundo transparente, aparecerão.

Quando eu uso a url de uma imagem salva no meu computador, o fundo não aparece, mas quando uso o link dessa mesma imagem hospedada em um site funciona.

Você provavelmente está fazendo alguma coisa errada na hora de colocar o endereço da pasta onde está a imagem.

Caso eu queira sobrepor uma imagem. Única forma que consegui, foi inserindo a imagem pelo arquivo index em html. Exemplificando: -----Imagem de fundo aplicada em CSS--- para sobrepor essa imagem com outra menor, consigo adicionando a imagem no arquivo HTML. Gostaria de saber se consigo fazer o mesmo utilizando o CSS?​​​​​​​​

Você ainda está muito no começo para fazer algo do tipo. Nas aulas posteriores você aprenderá alguns conceitos que poderão te ajudar com isso.

​Como eu formato uma imagem maior que o fundo no qual desejo que ela fique, para ela se "adequar" ao tamanho, mesmo que deforme. é possível fazê-lo?

Você pode usar o background-size: contain ou cover, onde um vai garantir que a imagem sempre apareça inteira (não ocupando todo o elemento) e o outro que sempre ocupe todo elemento, escalando a imagem conforme necessário. Depois pode usar background-position para definir onde a imagem começa, por exemplo ao colocar center, mesmo que a imagem não apareça inteira, vai sempre aparecer o centro dela no centro do elemento.

É possível em uma section, ou até mesmo em uma div, eu colocar uma background image e deixar a imagem sendo usada totalmente sem cortes como background desta div ou section? Fui fazer alguns testes, e se eu coloco uma background image, ela fica cortada dentro da section, só aparece parte da imagem e só aparece mais da imagem se eu aumento o conteúdo. E se eu uso o background-size seja contain ou cover a imagem nunca aparece inteira.

Então, ao usar background-size: contain ele irá sempre aparecer toda a imagem, porém com isso ele poderá não ocupar todo o conteúdo do elemento porque ele força a iamgem a aparecer mas mantém a proporção da imagem. Vou deixar a documentação sobre https://developer.mozilla.org/pt-BR/docs/Web/CSS/b...

Como faço pra deixar o background-image transparente? tentei usar o opacity: 0.5 mas n funcionou. quero deixar a imagem de fundo colocada no body transparente pra n atrapalhar a leitura.

Você não vai conseguir aplicar uma opacidade via propriedades em uma imagem de fundo. Seria possível colocar uma div por cima para ter esse efeito, mas exigiria tantos ajustes que nesse caso o mais indicado seria editar a imagem para que ela já fique do jeito que você quer. Esse vai ser o jeito mais simples de chegar ao resultado desejado.

Quando você adiciona um background para os <p>, a cor ou imagem adicionada vai até o final da página/navegador; é possível que o background adicionado fique do tamanho (largura) exata do texto (parecido como acontece com o bg-color da tag mark)?

A largura padrão de um elemento parágrafo é 100%. Você pode então alterar esta largura para um valor desejado, assim o background vai respeitar esse valor, ou você pode mudar o comportamento padrão de display do parágrafo com a propriedade display. Veremos essa propriedade para frente.

Não entendi a função do atributo ''local'' em attachment, qual seria a funcionalidade dele?

O valor "local" faz com que ele fique fixo somente referente ao elemento que foi aplicado. Você verá que em alguns momentos quando fixamos com um height um elemento, mas o seu conteúdo é maior do que aquele valor, o conteúdo acaba vazando, e utilizamos uma propriedade chamada overflow para evitar isso, com isso pode ser gerado um scroll de rolagem. Essa rolagem interna fará com que a imagem se mova também.

Dá uma olhada aqui na documentação: https://developer.mozilla.org/pt-BR/docs/Web/CSS/b...​

​​​​​​​​​​​​​​​​​​​​​​​Não compreendi direito o atributo "background-attachment" e suas variações (fixed,scroll e local). Poderia explicar e comentar a diferença entre eles?​

Tanto o scroll tanto o local, vão se basear no elemento em si, então se por ventura você tiver uma div com uma imagem de fundo, e essa div é limitada por exemplo por 500px de altura, mas o conteúdo acaba vazando. Quando você usar o overflow: auto, vai aparecer uma barra de rolagem. A sua imagem em si não vai se mexer em relação ao contexto que se encontra, quando você rolar o conteúdo interno ela continuará fixa, porém se ainda fora desse contexto ainda aja outros elementos e você role a viewport, a sua imagem vai ficar no seu ponto inicial e não acompanhará a rolagem externa. Diferente do caso do fixed, que mesmo que eu role a página, a imagem ficará fixa em certa parte da sua página. Vou deixar dois artigos para que você possa dar uma lida: https://developer.mozilla.org/pt-BR/docs/Web/CSS/b... e https://www.w3schools.com/css/css_background_attac...

Qual a diferença do background para o background-color?

O background sozinho, você pode colocar varios argumentos dentro dele, simplificando o código. 

Exemplo: 

background-image: url("imagem"); 

background-repeat: no-repeat; 

background-attachment: fixed; 

então você para simplificar, você poderia colocar assim: 

background: url("imagem") no-repeat fixed;​​
