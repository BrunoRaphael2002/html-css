​Lembre-se de se certificar de alguns pontos: 

1 - sempre cria a nome dos arquivos em minusculo, caso vc escreva Style.css e tente importar style.css pode acontecer de não carregar. 

2 - é importante reparar aonde está sendo criado o arquivo CSS, se vc criar uma pasta e colocar o arquivo dentro da pasta, vai precisar acessa a pasta e o arquivo no import, senão o arquivo não vai ser encontrado. 

3 - não pode esquecer de fazer o import do css no topo do html ali depois do title da pagina por exemplo, se não fizer o import não vai carregar o CSS. <link rel="stylesheet" href="styles.css"> styles.css sera substituído pelo nome que deu ao arquivo.

​No caso, a partir do momento que criamos o CSS Externo, ele tem valor superior ao CSS Interno, certo? Ou caso isso poderia gerar conflito com 2 informações.

Não necessariamente, ai nós entramos em especificidade no css, vou deixar dois links para vc tentar entender mais sobre, existe um calculo que determina o quão forte é um seletor CSS, quando vc passa o mouse em cima de um seletor em um doc .css ele mostra qual a especificidade daquele seletor, por exemplo, um seletor de classe tem um peso (0, 1, 0), já um seletor de id(#) tem um peso (1, 0, 0), a ordem da declaração faz toda diferença no css, é por isso que é importante seguir a mesma ordem hierárquica do html, no css o ultimo código é o que vai valer, mas esse comportamento muda de acordo com a especificidade de cada seletor, uma cor de fundo mudada por um seletor de id e depois por um de classe, seguindo a logica do peso de cada um o valor da classe não vai substituir o do id. 

Resumindo, vai valer o mais específico. Quando você cria um estilo que serve para qualquer parágrafo, esse estilo vai ser sobrescrito pelo inline pois o inline é aplicado a um único elemento. Ou seja é mais específico.

​Dando uma fuçada em um site de uma empresa de games, eu percebi que eles tinha um estilo lá para o site mesmo normal, e uma folha de estilo só para o formato mobile, (porque o site é tipo um app que você gerencia seu jogo). Minha dúvida: Essa ideia de separar os estilos css de website e mobile é alguma boa prática ? ou eles devem ter feito só pra facilitar mesmo organização? Eu colocar toda a minha formatação (site e mobile) num arquivo só é considerado uma boa prática?

​Então, é um pouco complexo isso, quando vc cria mais de um arquivo isso vai impactar um pouco na performance, agora vc tem dois arquivos para importar, precisa fazer duas requisições. Esse tipo de performance​ é algo que realmente faz uma diferença quando vc alcança determinado trafego no seu site, para sites de um iniciante não vai fazer diferença, isso são abordagens mais avançadas que não vamos lidar tanto aqui no curso, mas existem ferramentas que vc cria vários arquivos separadas e elas juntam tudo em um arquivo só no final, para ter uma melhor performance​, o ReactJS que vamos ver aqui no curso faz isso, então o ideal mesmo seria usar esse tipo de ferramenta. De inicio, não tem problema vc separar em dois arquivos se isso te ajudar a ser mais produtivo.​

​Com o style externo seria uma boa pratica criar para cada página do meu site uma folha de estilo?

​Não porque dessa forma vc vai acabar repetindo muito CSS, vc ate pode ter uma folha pra cada pagina se o site for muito grande, mas o ideal é vc fazer na mesma folha e focar em criar códigos reutilizáveis, classes genéricas para não precisar ficar repetindo o mesmo código e acabar com uma folha com duas mil linhas
