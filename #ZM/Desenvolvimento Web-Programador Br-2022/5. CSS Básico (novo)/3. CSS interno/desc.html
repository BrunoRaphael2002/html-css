O arquivo que estamos trabalhando não é '.html'? Como é que uma atribuição CSS roda de boa? Pensei que funções de CSS só funcionassem em arquivos '.css'?

​O que acontece é o seguinte, o css é uma linguagem de estilização, quando o navegador lê o documento html ele já saberá o que fazer com certos comandos. O navegador já tem certa inteligência do que deve fazer quando visualizar certos comandos. Quando existem comandos CSS, o navegador vai ler e entenderá que deve mostrar certo elemento de maneira diferente. Quando temos um arquivo separado em .css, significa dizer que dentro só terá comandos css, mas de toda forma nos estamos importando todos aqueles comandos para dentro do html, então meio que tudo que for escrito ali será colocado no documento no final das contas. Recomendo dar uma pesquisada na história do html, css e js. Além de dar uma lida aqui: https://developer.mozilla.org/pt-BR/docs/Learn/Get...

Pode usar o CSS interno tanto no head quanto no body ?? Eu acabei usando no body so para testar e funcionou mas não sei se e uma prática correta a se fazer.

Para muitas coisas ele irá funcionar corretamente sim, porém como o código é lido de cima pra baixo, o correto é o css já ser importado antes do body sempre.

É recomendado no head porque quando você tiver o CSS declarado antes <body> início, seus estilos já foram carregados. Muito rapidamente, os usuários veem algo aparecer na tela (por exemplo, cores de fundo). Caso contrário, os usuários verão a tela em branco por algum tempo antes que o CSS chegue ao usuário. Além disso, se você deixar os estilos em algum lugar no <body>, o navegador precisará renderizar novamente a página provavelmente vai carregar de novo.

Pelo q entendi o CSS inline e interno são utilizados em situações bem específicas? Geralmente devemos usar o externo para uma melhor organização, interpretação do código e por ser uma boa prática? Caso eu utilize o CSS externo, teria problema eu usar o inline ou interno em conjunto com ele? ou é melhor sempre optar por usar apenas 1 dos 3 em meus projetos?

O ideal é que padronize o código, se está utilizando externo, fique apenas com ele, mas não há nenhum problema em utilizar css externo em conjunto com interno e/ou inline. Entretanto, nesse ponto de vista, isso pode gerar comportamentos inesperados dependendo da forma que esteja fazendo, pois ele iria substituir algumas propriedades que aplicou. O mais interessante seria separar em um CSS externo para você poder ter uma melhor leitura futura e saber onde cada coisa está.​​​​​​​ ​​

​Caso eu pegue um código de um cliente e esteja assim eu posso mudar e deixar como é usado agora no HTML-5?

O código em HTML5 só vai funcionar se vc atualizar o doctype da página para ser compatível com o HTML5, e isso pode fazer com que vc tenha que realizar outras alterações na pagina, pode ser que nada pare de funcionar mas se vc fizer isso é bom garantir e dar uma refatorada em todo o código da pagina

Parece que o CSS inline é mais facil do que o CSS interno.​​​​​​​

No inline, você precisaria aplicar cada estilo direto em cada elemento, fazendo com que seu código em algum momento fique muito cheio e se torne confuso. Você vai vê o uso do CSS externo e com o tempo isso vai ficando mais fácil.
