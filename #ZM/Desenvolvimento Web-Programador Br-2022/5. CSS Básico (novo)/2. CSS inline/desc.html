DÚVIDAS DOS ALUNOS

​É visto como uma má prática usar o CSS inline? Pergunto isso pois às vezes parece ser bem conveniente usar ele para alterar um tag específica, sem ter que apelar para ID's adicionais.

O grande problema do CSS inline é que é muito fácil você perder o controle. Tecnicamente ele não é um problema em si, mas a gente sempre vai tentar criar grupos de elementos que compartilhem estilos em comum. Por exemplo, você tem dez botões na página e você quer que todos sigam o mesmo estilo. Com CSS inline isso vai ser um risco de você acabar modificando algo ali, outra coisa aqui e ter botões inconsistentes. E quando isso acontece, dependendo do tamanho da página, se perde muito tempo buscando onde está o erro.

O ";" é realmente necessário? Fiz sem ele no final e funcionou igual quando coloco.

No CSS ele vai ser obrigatório para separar uma propriedade da outra. Caso seja sua única ou última propriedade ai ele é opcional pois não vem nada depois para separar.

É necessário dar espaço depois dos dois pontos " : " ou é opcional. Fiz sem o espaço e o resultado foi o mesmo.

​Se dá o espaço apenas por questão de indentação (Organização do código)

Qual a importância do ";" (ponto e virgula) no final da tag?

O ; serve para fazer a divisão sobre cada estilo adicionado de maneira inline, na verdade no css é uma coisa muito importante, sem ele seu código pode tomar um resultado inesperado, ele basicamente significa o fim daquela atribuição, a partir dele ocorre uma nova atribuição de estilo.

Como assim ? Quando colocamos o color: blue; o ; significa que é o fim da atribuição e o que tiver depois seria considerado como fim do bloco ou uma nova atribuição. ex: color:red; background:green;
